# Human Test Checklist: PyPI + src/ migration (v0.8.x)

**Target version:** 0.8.4 (the docs/checklist commit of the 0.8.x arc)
**Ships with commits:** 0.8.0 (packaging metadata), 0.8.1 (cp1252-safe CLI),
0.8.2 (src/ layout + autobump path fixes), 0.8.3 (trusted-publishing
`release.yml`), 0.8.4 (README + this checklist).
**Companion automated tests:** 197 pytest tests (full suite), incl.
`tests/test_console_safe.py` (cp1252 fallback, red-green verified).

## What this is

A capstone checklist for shipping folder-datetime-fix to PyPI as a
cross-platform package. The automated suite + the clean-venv pre-flight cover
a lot, but this document covers the surfaces they can't: real `pip install`
resolution from PyPI, the `fdtfix` console script + the no-install shim, the
CLI on a legacy Windows code page, and the trusted-publishing release flow.

## Prerequisites

1. **OS/tooling:** Windows (primary). For the matrix: `uv` (Pythons 3.9-3.12)
   and WSL with Python 3.9+. A throwaway venv for every install test -- never
   `pip install` into your daily environment.
2. **Reset between sections:** install tests use fresh venvs; delete after.
   The pre-flight makes + removes its own venv in `%TEMP%`/`/tmp`.

## High-Value Verification (~5 minutes)

### HV.1 -- Clean-venv pre-flight GREEN (whole arc in one command)
Validates: src-layout editable install + dynamic version + full suite.
- [ ] **Steps:** `python tests/one-offs/preflight_clean_venv.py`
- [ ] **Expected:** ends `[preflight] GREEN`, `197 passed, 1 skipped`; the
  `pip show` block lists dazzle-filekit / dazzletreelib / dazzle-lib /
  pytest-asyncio installed in the throwaway venv.
- [ ] **Why this matters:** proves the src/ move didn't break install or tests.

### HV.2 -- Built wheel installs from a distribution + `fdtfix` runs
Validates: the actual distributable (not editable) + dep resolution from PyPI.
- [ ] **Steps:** in a throwaway venv:
  ```
  python -m build
  pip install dist/folder_datetime_fix-*.whl
  fdtfix --version
  fdtfix --help
  ```
- [ ] **Expected:** deps (dazzletreelib 0.10.x, dazzle-filekit, dazzle-lib,
  unctools) resolve from PyPI; `fdtfix --version` prints `fdtfix 0.8.x...`;
  `--help` shows `usage: fdtfix ...`.
- [ ] **Why this matters:** closest thing to `pip install folder-datetime-fix`.

### HV.3 -- All three console commands work
- [ ] **Steps (installed venv):** `fdtfix --version`, `folder-datetime-fix --version`, `mod_fldr_dt --version`
- [ ] **Expected:** all three print the same version (the `fdtfix` short alias +
  the two legacy names).

### HV.4 -- No-install shim runs from a git clone
Validates: `fdtfix.py` bootstraps `src/` so the tool runs without installing.
- [ ] **Steps:** in a **fresh clone** with NO install: `python fdtfix.py --version`
- [ ] **Expected:** prints the version (the shim adds `src/` to `sys.path`).
- [ ] **Why this matters:** the "git clone and just run it" path you asked to keep.

### HV.5 -- CLI does not crash on a cp1252 console
Validates: emoji/box-char output degrades to ASCII on legacy Windows code pages.
- [ ] **Steps (Windows cmd.exe):**
  ```cmd
  chcp 1252
  fdtfix . --dry-run -v
  fdtfix --help
  ```
- [ ] **Expected:** runs to completion, no `UnicodeEncodeError`; tree/help use
  ASCII markers (`[DIR]`, `+-- `) instead of emoji/box glyphs.
- [ ] **Why this matters:** the primary platform; this crashed before 0.8.1.

### HV.6 -- Cross-platform / cross-version
- [ ] **Steps:** `uv run --no-project --python 3.9 python tests/one-offs/preflight_clean_venv.py` ; `MSYS_NO_PATHCONV=1 wsl.exe python3 /mnt/c/code/modified_datetime_fix/local/tests/one-offs/preflight_clean_venv.py`
- [ ] **Expected:** both `[preflight] GREEN`.

## Section 1: Packaging & metadata

### 1.1 sdist/wheel contents are clean
- [ ] **Steps:** `python -m build` then inspect: `tar tzf dist/*.tar.gz` and the wheel's namelist.
- [ ] **Expected:** wheel = `folder_datetime_fix/` (all subpackages: help/, help_lib/) + dist-info, nothing else. sdist top-level = LICENSE, README, pyproject.toml, src/, tests/ -- **no** `private/`, `config/`, `revisions/`, `README2.md`.

### 1.2 Metadata
- [ ] **Check:** `pip show folder-datetime-fix` -> `Requires: dazzletreelib, dazzle-filekit`; classifier `OS Independent`; `Requires-Python: >=3.9`. Entry points: `fdtfix`, `folder-datetime-fix`, `mod_fldr_dt`.

### 1.3 Editable install works without setup.py
- [ ] **Steps:** fresh venv -> `pip install -e .` -> `python -c "import folder_datetime_fix"`
- [ ] **Expected:** succeeds via PEP 660 (no setup.py; setuptools>=64).

## Section 2: System-file litter (cross-platform)

### 2.1 Mac/Linux litter recognised
- [ ] **Check:** `src/folder_datetime_fix/system_files.py` includes macOS
  (`.DS_Store`, `._*`, `.Spotlight-V100`, `.fseventsd`, `.Trashes`) and Linux
  (`.directory`, `.hidden`) entries; the README "What counts as system litter"
  section matches.

### 2.2 `.git` default behaviour
- [ ] **Check:** `.git`/`.svn`/`.hg` are excluded by default (VCS metadata),
  but `.gitignore` is NOT. (Configurable opt-out is a Phase B follow-up.)

## Section 3: Release flow (trusted publishing) -- maintainer only

### 3.1 release.yml shape
- [ ] **Check:** `.github/workflows/release.yml` triggers on `release: published`;
  publish job has `environment: pypi` + `permissions: id-token: write` and uses
  `pypa/gh-action-pypi-publish`. Matches the PyPI trusted-publisher config
  (repo DazzleTools/folder-datetime-fix, workflow `release.yml`, env `pypi`).
- [ ] **Repo settings:** a GitHub Actions environment named `pypi` exists.

### 3.2 Publish (the real thing, once)
- [ ] **Steps:** merge dev -> main (clean version), tag `v0.8.x`, create a
  GitHub Release -> `release.yml` runs build + publish.
- [ ] **Expected:** the run's publish job succeeds (OIDC, no token); the version
  appears on https://pypi.org/project/folder-datetime-fix/ .
- [ ] **Then:** `pip install folder-datetime-fix` in a clean venv on a clean box
  -> `fdtfix --help`.

## What the automated tests DON'T cover

- **`pip install` resolution from PyPI** (deps actually pulled) -- HV.2.
- **cmd.exe cp1252 rendering** of the real CLI -- HV.5.
- **The no-install shim** from a fresh clone -- HV.4.
- **The trusted-publishing release** -- Section 3 (maintainer, once).
- **Cross-OS install** on real macOS/Linux hardware (CI covers Linux; mac via CI).

## Reset / cleanup

- The pre-flight removes its own venv. Delete any manual throwaway venvs and the
  local `dist/` + `build/` + `*.egg-info/` after build tests.
- No repo state is modified by any step.
